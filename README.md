## 📌 Огляд проекту
Комплексна клієнт-серверна система, розроблена для автоматизації процесів технічної підтримки на підприємстві. Система дозволяє користувачам швидко подавати заявки про несправності, автоматично збираючи технічний контекст (стан мережі, навантаження CPU/RAM) та передаючи ці дані адміністратору через захищене REST API.



## 🚀 Ключові можливості
* **Автоматизований збір даних:** Визначення Hostname, IP-адреси та поточного користувача Windows без ручного введення.
* **Модуль первинної діагностики:** * Перевірка мережевої доступності (Ping) шлюзу та серверів.
    * Збір метрик ресурсів (навантаження CPU, вільна RAM, залишок місця на диску) за допомогою бібліотеки `psutil`.
* **Інтерактивна адмін-панель:** Веб-інтерфейс на базі Bootstrap для моніторингу заявок у реальному часі.
* **Централізована БД:** Надійна архітектура на базі PostgreSQL.
* **Гнучке логування:** Збереження скріншотів помилок для візуального аналізу інцидентів.

## 🛠 Технологічний стек
- **Мова програмування:** Python 3.10+
- **Backend:** FastAPI (асинхронний фреймворк)
- **Database:** PostgreSQL (реляційна БД)
- **Frontend (Admin):** HTML5, Jinja2 templates, Bootstrap 5
- **Діагностика:** `psutil`, `subprocess` (системні виклики)
- **Мережа:** `requests`, `uvicorn`

 ## Безпека та надійність
- API Security: Усі запити від клієнта до сервера валідуються через X-API-Key у заголовках HTTP.

- Environment Variables: Використання .env запобігає витоку конфіденційних даних бази даних.

- Error Handling: Реалізовано обробку таймаутів та виключень при роботі з мережею та БД.

## Порядок роботи з програмою
Запустіть клієнтський додаток.

Введіть короткий опис проблеми (наприклад, "Не працює кнопка 2").

Оберіть категорію системи (РТГ, Інше тощо).

Натисніть "Відправити заявку".

Адміністратор миттєво побачить ваше звернення разом із результатами діагностики в адмін-панелі за адресою 127.0.0.1:8000/admin.

## 👩‍💻 Посібник для швидкого старту (Developer Guide)

### 1. Системні вимоги
* **Python 3.10+** (скачати з python.org)
* **Git** (скачати з git-scm.com)
* **PostgreSQL 14+** (або Docker Desktop)

### 2. Клонування проєкту
Bash
git clone [https://github.com/StasBursuk/krb_project.git](https://github.com/StasBursuk/krb_project.git)
cd krb_project 

### 3. Налаштування середовища
Створіть віртуальне оточення та встановіть залежності:

Bash
python -m venv venv
# Windows:
.\venv\Scripts\activate
# Linux/macOS:
source venv/bin/activate

pip install -r requirements.txt
### 4. Налаштування бази даних
Створіть БД у PostgreSQL (наприклад, it_support_db).

Скопіюйте файл .env.example у .env та вкажіть свої дані:
DATABASE_URL=postgresql://user:password@localhost:5432/it_support_db

### 5. Запуск проєкту
Запуск серверної частини (API):

Bash
uvicorn backend.main:app --reload
Запуск клієнтського додатку:

Bash
python client/report_tool.py
