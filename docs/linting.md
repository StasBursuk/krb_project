# Документація з якості коду (Linting & Static Analysis)

## Обрані інструменти
1. **Pylint**: Для статичного аналізу та перевірки відповідності стандарту PEP8.
2. **Black**: Для автоматичного форматування коду (безкомпромісний стиль).
3. **Mypy**: Для статичної типізації Python коду.

## Налаштування
- Базові правила описані у `.pylintrc`.
- Максимальна довжина рядка: 100 символів.
- Ігноруються папки `venv/` та `__pycache__/`.

## Інструкція з запуску
1. Комплексна перевірка: `pylint backend/`
2. Перевірка типів: `mypy backend/`
3. Автоформатування: `black .`

## Git Hooks
Впроваджено `pre-commit` хуки. Код автоматично форматується та перевіряється перед кожним збереженням у репозиторій.
