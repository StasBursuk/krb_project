## 1. Методологія
- **Інструмент:** Pyinstrument для CPU, `psutil` для моніторингу пам'яті.
- **Опареційне середовище:** Windows 11, Python 3.10, PostgreSQL 14.

## 2. Результати профілювання
| Операція | Середній час (ms) | Вузьке місце |
| :--- | :--- | :--- |
| Авторизація | 120ms | Хешування пароля (bcrypt) |
| Подача заявки | 450ms | Завантаження скріншота (I/O) |
| Генерація звіту | 890ms | Складний SQL JOIN |

## 3. Виявлені проблеми
1. **N+1 Query Problem:** При завантаженні списку заявок БД робить окремий запит для кожного імені користувача.
2. **Memory Spike:** При обробці скріншотів > 5MB пам'ять зростає на 200MB через відсутність потокового читання.
3. **Blocking GUI:** Клієнтський додаток (Tkinter) "замерзає" під час відправки даних на сервер.

## 4. Рекомендації
- Впровадити асинхронне читання файлів (Aiofiles).
- Використовувати `eager loading` в SQLAlchemy для усунення N+1.
- Винести мережеві запити клієнта в окремий потік (threading).
